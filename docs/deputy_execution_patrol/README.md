# Deputy Execution Patrol

This folder is the dedicated handoff lane between the Executive Officer and Deputy Codex.

The Executive Officer may chase assigned workstreams and publish concise operational follow-ups, but Deputy Codex keeps final routing and merge authority.

Allowed Executive Officer actions:
- Chase assigned workstreams.
- Comment on assigned Issues / PRs.
- Publish `EXECUTIVE_FOLLOW_UP`, `EXECUTIVE_CALL_OUT`, or `EXECUTIVE_STATUS_CHECK` entries to `docs/WORKSTREAM_BLACKBOARD.md`.
- Append decision requests to `EXECUTIVE_PATROL_INBOX.md`.

Not allowed:
- Merge / reject PRs.
- Close / reopen Issues.
- Create final new formal dispatches outside the active Issue scope.
- Decide final `Need Commander` / `Need Reviewer`.
- Reassign cross-workstream ownership.
- Touch secrets, payment, formal pricing, formal Excel/PDF, real AI API, upload, or destructive git operations.
