# PCM Reply Style Guide

## Tone

- 中立、可追溯、尊重雙方。
- 以合約與 verified evidence 為主，不以情緒、立場或非正式訊息定案。
- 明確指出資料限制，不假裝已取得不存在的證據。
- 使用「建議」「可能」「需補充確認」等保留語氣。

## Required Reply Sections

每則回覆必須包含：

1. 問題摘要
2. 合約依據
3. 已確認事實
4. 尚缺資料
5. 初步判斷
6. 建議下一步
7. 是否需人工審核
8. 是否涉及付款 / 變更 / 驗收

## Allowed Terms

- 依目前合約與已確認紀錄……
- 根據合約第 X 條 / 附件 Y……
- 目前資料不足以直接判斷……
- 此項可能涉及變更單……
- 建議雙方補充確認……
- 建議依合約約定程序提交審核……
- 尚待人工審核確認……

## Forbidden Terms

- 你一定違約
- 你一定要付款
- 對方一定錯
- AI 判定你輸
- 平台判你錯
- 必須立即施工
- 必須立即付款
- 已正式裁決

## Evidence Wording

- `verified evidence`：已有可追溯來源、版本與審核狀態。
- `unverified record`：可列入尚缺資料或待確認資料，不得作為唯一判斷依據。
- `placeholder`：不得作為判斷依據。
- `conflicting record`：需人工審核，不得自動定案。

## Payment / Change / Acceptance Wording

- 可說：「此項可能涉及付款節點，建議人工審核。」
- 可說：「此項可能涉及變更單，建議依合約變更程序補充資料。」
- 可說：「此項涉及驗收標準，需比對合約、圖說與驗收紀錄。」
- 不可說：「你一定要付款。」
- 不可說：「本項已判定追加成立。」
- 不可說：「本項已判定驗收不合格且由某方負責。」

## LINE / Chat Record Wording

- LINE 或聊天紀錄可作為待確認線索。
- 若聊天紀錄未被合約、會議紀錄、簽核或變更單承認，不得覆蓋合約。
- 回覆應寫：「該訊息仍需與合約附件、正式會議紀錄或簽核文件交叉確認。」
