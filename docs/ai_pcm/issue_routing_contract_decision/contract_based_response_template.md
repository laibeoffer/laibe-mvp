# Contract-Based Response Template

> 使用限制：本模板產出的是中立回覆草稿，不是正式法律裁決，也不是付款、施工或違約命令。

## 1. 問題摘要

以中立語氣摘要問題、提出方、涉及位置、工項、時間與文件版本。不得加入「誰對誰錯」的結論。

## 2. 合約依據

- 根據合約第 X 條 / 附件 Y / 圖面 Z / 會議紀錄 N。
- 若尚未取得正式條文，寫明：「目前尚未取得可驗證之合約條文，需補充合約本文或附件版本。」
- LINE 訊息、口頭說明或截圖僅可列為待確認資料，不得覆蓋合約。

## 3. 已確認事實

- 列出 verified evidence。
- 每項事實應包含來源、日期、版本與驗證狀態。
- 不確定資料不得放入本段，應放入「尚缺資料」。

## 4. 尚缺資料

- 列出完成判斷前必須補充的合約、圖面、照片、簽核、會議紀錄或變更單。
- 指明由哪一方或哪個系統補充，但不得命令付款或施工。

## 5. 初步判斷

使用保留語氣：

- 依目前合約與已確認紀錄……
- 目前資料不足以直接判斷……
- 此項可能涉及變更單……
- 建議雙方補充確認……

不得使用正式裁決或違約判定語氣。

## 6. 建議下一步

- 補充缺漏文件。
- 指定需確認的合約條款或附件。
- 建議召開澄清或人工審核。
- 若涉及變更，建議依合約變更程序處理。

## 7. 是否需人工審核

格式：

```text
需人工審核：是 / 否
原因：涉及付款 / 變更 / 驗收 / 違約主張 / 證據衝突 / 合約條款解釋 / 資料不足。
審核對象：AI PCM 總監／後台總控 Agent 或指定 PCM 人員。
```

## 8. 是否涉及付款 / 變更 / 驗收

格式：

```text
涉及付款：是 / 否 / 尚待確認
涉及變更：是 / 否 / 尚待確認
涉及驗收：是 / 否 / 尚待確認
```
